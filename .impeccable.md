## Design Context

### Users
Technical users, developers, and administrators. The primary context is editing XML data structures quickly and precisely. The interface should feel utilitarian, fast, and dense to allow for high productivity without unnecessary friction.

### Brand Personality
Technical, minimalist, precise, and utilitarian. It should evoke feelings of control, speed, and confidence in making changes to critical data.

### Aesthetic Direction
Dark mode only. The visual tone should be focused on the content (XML structure and editing) with high contrast for readability but reduced visual noise. It should avoid overly playful or generic "AI" aesthetics (no gradients, no rounded corners everywhere), leaning into a "developer tool" or "terminal-inspired" aesthetic but modernized.

### Design Principles
1. **Information Density over Whitespace**: Prioritize showing more data and structure at a glance rather than padding everything out.
2. **Keyboard-First Affordances**: Visual indicators should suggest speed and keyboard navigability (even if not fully implemented yet).
3. **Muted Color Palette**: Rely on typography, contrast, and subtle borders rather than bright colors to define hierarchy. Reserve bright colors strictly for validation errors or critical actions.
4. **Monospace & Precision**: Use monospace or technical sans-serif fonts to align with the codebase/XML editing nature of the tool.
